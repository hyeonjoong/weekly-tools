# 사용법 (빠른 시작)

사용자 이벤트 로그 CSV → **세션 · DAU/WAU/MAU · 리텐션 · 퍼널**을 한 번에 요약합니다.

## 가장 쉬운 방법
- 이 폴더의 **`실행.command`** 파일을 더블클릭하세요.
- 번들 예시(수면앱 사용 로그 6명)로 분석이 자동 실행되어 결과를 보여줍니다.

## 내 데이터로 돌리기 (터미널)
CSV에 **사용자/이벤트/시각** 세 열이 있으면 됩니다. (기본 열 이름: `user_id`, `event`, `timestamp`)

```bash
# 1) 기본 실행
python3 -m logflow.cli 내로그.csv

# 2) 퍼널까지 보기 (단계는 순서대로)
python3 -m logflow.cli 내로그.csv --funnel app_open,breathing_start,breathing_complete

# 3) 열 이름이 다를 때 + 리텐션 day-1/3/7
python3 -m logflow.cli 내로그.csv \
    --user-col uid --event-col action --time-col ts --retention 1,3,7
```

타임스탬프는 `2026-01-01T22:00:00` 같은 ISO 형식이나 epoch 숫자 모두 됩니다.

## 결과 읽는 법
- **개요**: 총 이벤트/사용자, 기간, 세션 수, 세션당 평균 이벤트·길이.
- **활성 사용자**: 날짜별 DAU(당일)·WAU(7일)·MAU(28일). 맨 아래 점착도 = 평균DAU/평균MAU.
- **리텐션**: `day-1 : 50.0% (retained 3/6)` = 첫날 쓴 사람 6명 중 다음날 다시 온 3명 = 50%.
- **퍼널**: 위에서 아래로 단계별 도달 인원과 전환율(직전 단계 대비 / 1단계 대비). 어디서 이탈하는지 한눈에.
